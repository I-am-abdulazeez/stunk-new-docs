Stunk provides two ways to derive state from a single chunk — `.derive()` as a method directly on the chunk, and `select()` as a standalone function with more control over equality checking.

---

## `.derive(fn)`

The simplest way to create derived state. Call `.derive()` on any chunk with a transformation function — the result is a new reactive chunk that updates automatically.

```ts
import { chunk } from "stunk";

const price = chunk(100);

const discounted = price.derive((p) => p * 0.9);
const formatted = price.derive((p) => `$${p.toFixed(2)}`);

price.set(200);

discounted.get(); // 180
formatted.get(); // "$200.00"
```

The derived chunk is **read-only** — it updates when the source updates, but you can't set it directly. Modifying the source is the only way to change it.

### Chaining derives

Derived chunks can themselves be derived:

```ts
const count = chunk(10);
const doubled = count.derive((n) => n * 2);
const label = doubled.derive((n) => `Value: ${n}`);

count.set(5);
label.get(); // "Value: 10"
```

### Cleanup

A derived chunk stays subscribed to its source as long as both exist. If you destroy the **source chunk**, the derived chunk stops receiving updates automatically.

If you want to stop a derived chunk from updating while keeping the source alive, call `destroy()` on the derived chunk — this unsubscribes it from the source and frees memory:

```ts
const discounted = price.derive((p) => p * 0.9);

// Stop listening to price, but price still works fine
discounted.destroy();
```

<Callout type="info">
  In React, hooks like `useDerive` handle cleanup automatically when the
  component unmounts. Manual `destroy()` is only needed when using derived
  chunks outside of React.
</Callout>

## `select()`

`select()` is a standalone function that creates a read-only derived chunk from a source chunk. It gives you more control than `.derive()` — specifically, **shallow equality checking** to prevent unnecessary updates when deriving objects.

```ts
import { chunk } from "stunk";
import { select } from "stunk";

const user = chunk({ name: "Fola", age: 25, role: "admin" });

// Only re-renders when `name` changes — ignores age and role changes
const userName = select(user, (u) => u.name);

userName.get(); // "Fola"
```

### Shallow equality

By default, `select()` uses strict reference equality (`!==`) to detect changes. For object selectors, this means a new object reference always triggers an update — even if the contents are the same.

Enable `useShallowEqual` to avoid unnecessary updates when the selector returns a new object with the same values:

```ts
const user = chunk({ name: "Fola", age: 25, scores: [10, 20] });

// Without shallow equal — updates every time user changes
const basicInfo = select(user, (u) => ({ name: u.name, age: u.age }));

// With shallow equal — only updates when name or age actually changes
const basicInfo = select(user, (u) => ({ name: u.name, age: u.age }), {
  useShallowEqual: true,
});
```

### Read-only enforcement

`select()` enforces read-only access. Calling `set()` or `reset()` on a selector chunk throws an error:

```ts
const userName = select(user, (u) => u.name);

userName.set("Tunde"); // ❌ throws: "Cannot set values directly on a selector"
userName.reset(); // ❌ throws: "Cannot reset a selector chunk"
```

To update the value, modify the source chunk instead:

```ts
user.set((prev) => ({ ...prev, name: "Tunde" }));
userName.get(); // "Tunde"
```

## `.derive()` vs `select()`

|                  | `.derive()`       | `select()`                                   |
| ---------------- | ----------------- | -------------------------------------------- |
| Syntax           | Method on chunk   | Standalone function                          |
| Read-only        | Yes               | Yes                                          |
| Shallow equality | No                | Yes (`useShallowEqual`)                      |
| Chainable        | Yes               | Yes (chains return new selectors)            |
| Use case         | Simple transforms | Object slices, performance-sensitive derives |

Use `.derive()` for simple transformations. Use `select()` when you're extracting a slice of an object chunk and want to avoid unnecessary re-renders.

---

## In React

Use `useDerive` to consume a derived value reactively in a component:

```tsx
import { chunk } from "stunk";
import { useDerive } from "stunk/react";

const price = chunk(100);

function DiscountedPrice() {
  const discounted = useDerive(price, (p) => p * 0.9);

  return <p>Discounted: ${discounted}</p>;
}
```
